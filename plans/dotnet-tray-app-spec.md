# Спецификация .NET 10 Tray-приложения для мониторинга CO2

## 1. Обзор

Создать .NET 10 (Windows Desktop) приложение для Windows 10/11, которое будет работать в системном трее и отображать текущее состояние CO2 и температуры с сервера Go backend.

**Требования к иконке:**
- Цветная стрелка нужного направления (зеленый/оранжевый/красный) для отображения состояния CO2
- Серый кружок без стрелки для отображения отсутствия соединения с сервером
- Поддержка масштабирования DPI (16x16, 32x32 и т.д. для разных значений масштаба)

## 2. Требования

### 2.1 Основные функции

- **Трей-иконка** - отображение состояния CO2 с помощью нарисованной стрелки нужного цвета
- **Tooltip** - при наведении курсора показывает текущие значения CO2 и температуры
- **Контекстное меню** - Exit, Settings (открыть файл настроек)
- **Автоматический опрос** - периодическое получение данных с сервера

### 2.2 Цвета индикатора

| Состояние | Цвет | Условие |
|-----------|------|---------|
| Зеленый | Green | CO2 ≤ NormalThreshold (по умолчанию 1000 ppm) |
| Оранжевый | Orange | NormalThreshold < CO2 ≤ WarningThreshold (по умолчанию 1500 ppm) |
| Красный | Red | CO2 > WarningThreshold |
| Серый | Gray | Сервер недоступен (ошибка подключения) - простой кружок без стрелки |

### 2.3 Стрелки изменения

| Стрелка | Unicode | Условие |
|---------|---------|---------|
| Вверх-вправо (↗) | U+2197 | CO2 увеличилось по сравнению с предыдущим значением |
| Горизонтальная (→) | U+2192 | CO2 не изменилось |
| Вниз-вправо (↘) | U+2198 | CO2 уменьшилось по сравнению с предыдущим значением |

> **Примечание:** Стрелка всегда направлена вправо (горизонтально или по диагонали), чтобы соответствовать требованиям: "стрелка вправо" (не менялось), "стрелка в правый верхний угол" (растут), "стрелка в правый нижний угол" (падают).

### 2.4 Настройки (appsettings.json)

```json
{
  "ServerUrl": "http://localhost:8072",
  "PollingIntervalSeconds": 60,
  "NormalThreshold": 1000,
  "WarningThreshold": 1500
}
```

### 3.2 Компоненты

#### 3.2.1 Settings (`Settings.cs`)

```csharp
public class Settings
{
    public string ServerUrl { get; set; } = "http://localhost:8072";
    public int PollingIntervalSeconds { get; set; } = 60;
    public int NormalThreshold { get; set; } = 1000;      // ≤1000 ppm
    public int WarningThreshold { get; set; } = 1500;     // 1001-1500 ppm
}
```

#### 3.2.2 SettingsManager (`SettingsManager.cs`)

- Загрузка настроек из `appsettings.json` (файл рядом с приложением)
- Сохранение настроек в `appsettings.json`
- Путь к файлу: `./appsettings.json` (в корне проекта/приложения)

#### 3.2.3 DataClient (`DataClient.cs`)

**Методы:**
- `GetLatestReadingAsync()` - получить последнее значение CO2 и температуры
- `IsServerAvailableAsync()` - проверка доступности сервера

**API Endpoints:**
- `GET /api/data/latest?count=1` - получить последнее значение

**Response:**
```json
[
  {
    "timestamp": "2024-01-15T10:30:00Z",
    "temperature": 22.5,
    "co2": 450
  }
]
```

#### 3.2.4 IconManager (`IconManager.cs`)

**Поля:**
- `previousCo2Value` - хранит предыдущее значение CO2 для определения направления стрелки

**Методы:**
- `CreateIcon(current, previous)` - создать иконку с текущим состоянием
- `CreateDisconnectedIcon()` - создать серую иконку (сервер недоступен) - простой кружок без стрелки
- `GetArrowDirection(current, previous)` - определить направление стрелки по изменению CO2

**Логика:**
1. Определить направление стрелки по изменению CO2 (↗ рост, → стабильно, ↘ падение)
2. Определить цвет по текущему CO2 (зеленый/оранжевый/красный)
3. Если сервер недоступен - использовать серый цвет
4. Создать Bitmap 16x16 и нарисовать стрелку нужного цвета и направления
5. Преобразовать Bitmap в Icon для NotifyIcon

#### 3.2.5 TrayApp (`TrayApp.cs`)

**Поля:**
- `NotifyIcon` - значок в трее
- `Timer` - периодический опрос (настраиваемый интервал)
- `ContextMenuStrip` - контекстное меню
- `previousCo2Value` - хранит предыдущее значение CO2 для определения направления стрелки (сбрасывается при перезапуске, начальное значение = NormalThreshold)

**Методы:**
- `LoadSettings()` - загрузить настройки
- `FetchData()` - получить данные с сервера
- `UpdateTrayIcon()` - обновить иконку (вызывает IconManager.CreateIcon())
- `ShowTooltip()` - показать tooltip
- `OnTimerTick()` - обработчик таймера (вызывает FetchData и UpdateTrayIcon())
- `ExitMenuItem_Click()` - обработчик выхода из приложения
- `SettingsMenuItem_Click()` - обработчик открытия настроек

**Концепция работы:**
1. При запуске загружаются настройки из SettingsManager
2. previousCo2Value инициализируется значением NormalThreshold из настроек (начальное значение для сравнения)
3. Запускается Timer с интервалом PollingIntervalSeconds
4. При срабатывании таймера вызывается FetchData()
5. Если данные получены успешно, UpdateTrayIcon() создает новую иконку через IconManager
6. previousCo2Value обновляется после каждого успешного запроса

## 5. Реализация иконки

### 5.1 GDI+ рисование

**Алгоритм создания иконки (с учетом DPI):**
1. Определить текущий DPI масштаб: `DpiScale dpi = TextRenderer.DpiScale;`
2. Рассчитать размер иконки: `size = 16 * dpi.DpiScaleX` (округлить до ближайшего стандартного размера: 16, 32, 48)
3. Создать Bitmap с рассчитанным размером
4. Очистить фон (прозрачный)
5. Нарисовать стрелку или кружок нужного цвета
6. Преобразовать Bitmap в Icon для NotifyIcon

**Размеры иконок по DPI:**
| DPI | Размер иконки |
|-----|---------------|
| 100% (1.0) | 16x16 |
| 125% (1.25) | 20x20 |
| 150% (1.5) | 24x24 |
| 200% (2.0) | 32x32 |
| 250% (2.5) | 40x40 |
| 300% (3.0) | 48x48 |

**Цвета (ARGB) - для всех размеров иконок:**
| Состояние | Цвет | ARGB |
|-----------|------|------|
| Зеленый | Green | (255, 0, 128, 0) |
| Оранжевый | Orange | (255, 255, 165, 0) |
| Красный | Red | (255, 255, 0, 0) |
| Серый (кружок) | Gray | (255, 128, 128, 128) |

**Стрелки (для цветных иконок):**
| Состояние | Направление | Описание |
|-----------|-------------|----------|
| Рост | ↗ | Стрелка направлена вверх-вправо |
| Стабильно | → | Стрелка горизонтальная |
| Падение | ↘ | Стрелка направлена вниз-вправо |

**Кружок (для серой иконки Disconnected):**
- Простой круг без стрелки
- Центр иконки
- Радиус ~6-7 пикселей (для 16x16)

### 5.2 Tooltip формат

Одна строка: `CO2: 450 ppm (стрелка направления) | Температура: 22.5°C`

## 6. Обработка ошибок

| Сценарий | Действие |
|----------|----------|
| Ошибка подключения к серверу | Показать серую иконку (кружок без стрелки), повторить через обычный интервал опроса |
| Неверный JSON ответ | Логировать ошибку, использовать предыдущее значение CO2 и иконку |
| Таймаут запроса | Логировать ошибку, показать серую иконку (кружок без стрелки) |
| Отсутствие данных от сервера | Логировать ошибку, показать серую иконку (кружок без стрелки) |

> **Примечание:** При ошибках подключения иконка должна отображаться серой (простой кружок без стрелки). При неверном JSON используется предыдущее значение CO2 и соответствующая иконка.

## 9. План реализации

1. **Создать проект** - WinForms .NET 10 (CO2DashboardTray.csproj)
2. **Настройки** - Settings.cs, SettingsManager.cs (файл `appsettings.json` рядом с приложением)
3. **DataClient** - HTTP клиент для API (GET /api/data/latest?count=1)
4. **IconManager** - создание иконок через GDI+ с поддержкой DPI (Bitmap 16x16, 32x32 и т.д., рисование стрелки или кружка)
5. **TrayApp** - основной класс приложения (NotifyIcon, Timer, ContextMenuStrip, инициализация previousCo2Value = NormalThreshold)
6. **Протестировать** - проверить все сценарии (подключение, ошибки, смена значений, DPI масштабирование)

## 10. Примечания

- Приложение должно быть **без GUI форм** - только NotifyIcon
- Настройки хранятся в файле `appsettings.json` рядом с приложением
- По умолчанию опрос каждые 60 секунд
- Пороги по умолчанию: Normal=1000 ppm, Warning=1500 ppm
- При перезапуске начальное значение CO2 для сравнения = NormalThreshold
- Поддержка масштабирования DPI (100%-300%)
